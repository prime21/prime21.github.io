XRQRS

Given an initially empty integer array (1-indexed) and some queries:

  • Type 0: Add the integer number x at the end of the array.
  • Type 1: On the interval L..R find a number y, to maximize (x xor y).
  • Type 2: Delete last k numbers in the array
  • Type 3: On the interval L..R, count the number of integers less than or equal to x.
  • Type 4: On the interval L..R, find the kth smallest integer (kth order statistic).

注意到操作序列可以描述为一棵操作树，所以操作序列是可持久化的。
对于查询1，是查询一个可持久化Trie
对于查询3，是查询权值线段树
对于查询4，是在权值线段树上二分

对于每一个查询，时间复杂度是$\log m + \log v$
时空间复杂度是$m \log m$的，甚至可以允许强制在线

HDU 4821

统计有多少个子串满足：
能分成$M$段长$L$的两两不相同的字符串

$M,L,|S| \le 1e^5$

注意到所有子串可以划分为$L$类，以起点$\mod L$划分。

需要保证复杂度的题。

先考虑对串做hash

对于同一类，两个子串之间的枚举其实是删除一个小段，添加后一个小段。
对于每一类
共有$|S| \over L$个 小段，可以用map统计有多少个符合题意的子串，复杂度是 $O( \frac{|S|}{L} \log M)$的。

总复杂度是$|S| \log M$的，复杂度是对的。

BZOJ 3083

对一棵有根树，支持如下操作

  • 1 修改树的根为$id$
  • 2 将一条链$l,r$上的所有节点覆盖为$v$
  • 3 子树查询最小值

树链剖分的换根问题

实际并不换根,我们记录当前的根为$root$，对于一个子树的查询$x$

  • case 1
    $root==x$，查询整棵树即可
  • case 2
    $lca(root,x) \ne x$
    此时修改和$x$的查询与其无关
  • case 3
    $lca(root,x) == x$
    此时修改是找到$x$的所有儿子中，包含$root$的那个

SEABAL

有$n$个盒子里面有气球。
有$k$次操作，每次删除一个盒子中的一个气球
有$m$个区间，每次操作后，询问有多少个区间全部为空。
$n \in [1,100000] $

每个区间询问被考虑成$S(R)-S(L-1)=0$的形式
那么当i所在的块被删除的时候，查询一下左端点在 ( L[i] , i] ，右端点在 [i,R[i] )的区间。
这样就是一个简单主席树了

hdu 4366

一棵树，每个点有两个值$p$和$v$，做如下子树查询$T(x)$

按照$p$从大到小，当做插入的时间序列，用dfs序将子树问题划归为线段问题。
对于每次查询，相当于查询子树内的最大的$v$
在$v$为权值,$p$为时间的主席树上二分一下，复杂度是$n \log n$的
