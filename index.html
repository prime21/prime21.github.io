five stage single cycle mips cpu(logisim)

发表于 2018-11-19

阶段分析：

  • 取指(IF):根据程序计数器(Program Counter)中的指令地址，从指令存储器中取出一条指令，同时，PC根据指令长度自动递增产生下一条指令所需要的地址（情况有二：直接$+4$，或者读取j型指令中经过适当的变化后的目标地址）。
  • 译码(ID):对取指令操作中得到的指令进行分析并译码，确定这条指令需要完成的操作，从而产生相应的操作码，用于驱动执行状态中的各种操作。
  • 执行(EXE):根据指令译码得到的操作控制信号，使用ALU具体地执行指令动作，然后转移到结果写回状态。
  • 访存(MEM):所有需要访问存储器的操作都将在这个步骤中执行，该步骤给出存储器的数据地址，把数据写入到存储器中数据地址所指定的存储单元或者从存储器中得到数据地址单元中的数据。
  • 写回(WB):指令执行的结果或者访问存储器中得到的数据写回相应的目的寄存器中。

部件设计

IFU 取指令单元

端口定义

信号 方向 描述
branch input 分支信号使能
jump input 跳转信号使能
Flag input ALU比较结果
reset input 全局的Reset信号
Instr[31:0] output 32位指令信号

功能定义
| 序号 | 功能名 | 描述 |
| :-: | :-: | :-: |
| 1 | 复位 | 全局复位信号，将PC清0 |
| 2 | 取指令 | PC 从 IM 中取出指令 |
| 3 | 计算下一条指令地址 | 如果不是跳转和分支，则直接+4，如果是跳转PC <- PC+4+BrAdd, 如果跳转，则PC = JumAdd |

GRF 通用寄存器组

端口定义

信号 方向 描述
clk input 时钟信号，多在其上升沿做出效应
reset input 复位信号，为1时将所有寄存器清零（异步复位）
RegWrite input 写使能信号，为1时才能写入数据
a1 input 5位的地址输入信号，指定32个寄存器中的一个，将其存储的数据读出到rd1
a2 input 5位的地址输入信号，指定32个寄存器中的一个，将其存储的数据读出到rd2
RegAddr input 5位的地址输入信号，指定32个寄存器中的一个作为写入的目标
RegData input 32位的写入数据
rd1 input 输出a1所指定的寄存器中的32位数据
rd2 input 输出a2所指定的寄存器中的32位数据

模块功能定义

序号 功能名 描述
1 复位 当复位信号有效时，所有寄存器清零
2 读数据 读出a1,a2地址对应的寄存器中存储的数据到rd1,rd2
3 写数据 当we有效且时钟上升沿来临时，将wd写入a3所对应的寄存器中

ALU 算术逻辑单元

为了让alu支持所需的多种算术逻辑操作，便于设计我们将ALUop分为2+4,前两位叫运算类型ALUtp，后四位叫运算函数ALUfc，我们记输入为$A$和$B$,输出为$Y$，用$[exp]$表示取真函数，当$exp$为真时，$[exp]=1$否则为$0$

ALUtp 功能
00 32位比较运算
01 32位算术运算
10 32位逻辑运算
11 32位位移运算

对于ALUtp=00
| ALUfc | ALufc[ 3 ] | ALufc[ 2 ] | ALufc[ 1 ] | ALufc[ 0 ] | 功能 |
| :-: | :-: | :-: | :-: | :-: | :-: |
| | 符号位 | 小于判别 | 等于判别 | 大于判别 | |
|X010|X|0|1|0|Y=[A = B] |
|0110|0|1|1|0|Y=[A ≤ B] (无符号)|
|0011|0|0|1|1|Y=[A ≥ B] (无符号)|
|0100|0|1|0|0|Y=[A < B] (无符号)|
|0001|0|0|0|1|Y=[A > B] (无符号)|
|X101|X|0|0|1|Y=[A ≠ B] |
|1110|1|1|1|0|Y=[A ≤ B] (有符号)|
|1011|1|0|1|1|Y=[A ≥ B] (有符号)|
|1100|1|1|0|0|Y=[A < B] (有符号)|
|1001|1|0|0|1|Y=[A > B] (有符号)|

对于ALUtp=01
ALUfc仅有最低位有效
|ALUfc[ 0 ]| 功能 |
|:-:|:-:|
|0| Y = A+B |
|1| Y = A-B |

对于ALUtp=10
记$ALUfc = \overline{abcd}$则$Y[i] = F_{abcd} (A[i],B[i])$。
F是一个真值表函数
|Y|A|B|
|:-:|:-:|:-:|
|d|0|0|
|c|0|1|
|b|1|0|
|a|1|1|

那么常见的各种位运算函数的ALUtp如下表
|位运算|ALUfc|
|:-:|:-:|
|and|1000|
|or|1110|
|xor|0110|
|nor|0001|

对于ALUtp=11
仅有两位有效，最低位判断移动方向，倒数第二位判断是算术操作还是逻辑操作。
|ALUfc| ALufc[ 3 ] | ALufc[ 2 ] | ALufc[ 1 ] | ALufc[ 0 ] | 功能 |
|:-:|:-:|:-:|:-:|:-:|:-:|
|XXX0| X | X | X | 0 | Y = B << A |
|XX01| X | X | 0 | 1 | Y = B >>> A (逻辑右移) |
|XX11| X | X | 1 | 1 | Y = B >> A (算术右移)|
